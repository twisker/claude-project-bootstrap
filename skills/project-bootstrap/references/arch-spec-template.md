## 3. CI/CD 流水线

### 流水线概览

| 触发条件 | 执行内容 | 部署目标 |
|---------|---------|---------|
| push to main/master | Lint + 测试 + 构建 + 自动部署 | staging |
| 手动触发（1人审批） | staging 已验证产物晋升 | gray |
| 手动触发（2人审批） | gray 已验证产物晋升 | prod |

### CI 阶段（自动）

1. **代码质量**：Lint + 类型检查 + 依赖漏洞扫描
2. **测试**：单元测试 + 集成测试
3. **构建**：前端构建 + 后端构建/镜像打包
4. **部署 staging**：自动部署 + 冒烟测试

### CD 阶段（手动）

- **灰度部署**：部署灰度 -> 冒烟测试 -> 观察期
- **生产部署**：部署生产 -> 滚动更新 -> 健康检查

## 5. 版本管理与发布

### 版本格式

`major.minor.patch`（如 `1.2.34`）

### 自动管理规则

| 版本段 | 更新方式 | 触发条件 |
|--------|---------|---------|
| patch | 自动（git hook） | 每次 git commit |
| minor | 人工脚本 | 人工调用 `scripts/bump-minor.sh` |
| major | 人工脚本 | 人工调用 `scripts/bump-major.sh` |
