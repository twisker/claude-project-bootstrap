# 验收标准登记表

此文件记载各模块的交付标准或验收标准。包括通用交付标准，以及各模块的专项验收标准。

---

## 通用交付标准

所有交付物必须满足以下标准：

### 代码质量

| 标准 | 要求 |
|------|------|
| Lint 检查 | 无 error，warning 数量 <= 5 |
| 类型检查 | 严格模式通过（如适用） |
| 测试通过 | 所有测试用例 100% 通过 |
| 测试覆盖率 | 核心逻辑 >= 80%，页面级 >= 70% |

### 功能完整性

| 标准 | 要求 |
|------|------|
| 需求覆盖 | PRD 中标注的功能均已实现 |
| 边界处理 | 异常路径和空状态有合理处理 |

### 性能

| 标准 | 要求 |
|------|------|
| API 响应 | P95 <= 500ms（普通接口） |
| {根据项目补充} | |

### 基础设施与部署

| 标准 | 要求 |
|------|------|
| 环境隔离 | staging/production 数据层不可互访 |
| 回滚时效 | 回滚操作 < 60 秒 |
| CI/CD | 主分支推送自动部署 staging，生产手动触发 + 审批 |
